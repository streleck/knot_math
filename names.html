Chapter 2

Names

The goal is to give names to each possible knot configuaration, but not just any name will do. The names need to be unique, systematic, meaningful. The names should tell me somthing about the knot. How about if they tell me EVERYTHING (everything important, at least) about the knot? Again, what seems to be important are the crossings, and where they occur.

Recipe: My idea was to make it so that a knot's name is a recipe of how to make the knot. That way if you had a knot, but gave me just the name, I could recreate (in essence) the very same knot. This would, then, mean that all relevant information is contained within that name. Let's start cooking up some knots!

Step 1: We'll build knots by starting with the basic, single loop, and add more crossings to it. Imagine that one end of the rope is tied to a boat and one end is free. Refining the Number System This is the type of ambiguity that could undermine our whole naming system! If a name is a recipe, it should tell us exactly how to make the knot. One recipe should not produce two different possible knots. To settle this, we need to provide a bit more information: which side of the rope we are crossing. To resolve this, one side will be numbered, the other side will be lettered.
